## 1. Общие сведения о программе

### 1.1. Назначение программы

Программа предназначена для обеспечения функционирования официального веб-сайта кафедры информационных и коммуникационных технологий **федерального государственного бюджетного образовательного учреждения высшего образования «Российский государственный педагогический университет им. А. И. Герцена»** (РГПУ им. А. И. Герцена).

**Основные функции программы:**

| № | Функция | Описание |
|---|---------|----------|
| 1 | Публикация информационных материалов | Размещение новостей, объявлений, анонсов мероприятий |
| 2 | Предоставление учебных материалов | Доступ к лекциям, методическим пособиям, презентациям |
| 3 | Информирование абитуриентов | Сведения о направлениях подготовки (бакалавриат, магистратура), правилах приёма |
| 4 | Отображение научной деятельности | Информация о публикациях, конференциях, научных проектах |
| 5 | Обеспечение обратной связи | Приём заявок, опросов, обращений через веб-формы |
| 6 | Интеграция с Google-сервисами | Авторизация через Google OAuth, импорт данных из Google Forms, выдача файлов с Google Drive |
| 7 | Разграничение прав доступа | Аутентификация пользователей, суперпользователь с полными правами |
| 8 | Публикация мультимедийного контента | Фотогалереи, слайдеры, видеоматериалы |

### 1.2. Технические средства

Для выполнения программы требуется следующее техническое обеспечение:

Для выполнения программы необходимо следующее программное обеспечение:

| Компонент | Наименование | Версия |
|-----------|--------------|--------|

## 2. Структура программы

### 2.1. Общие сведения о структуре

Программа представляет собой веб-сайт кафедры информационных и коммуникационных технологий **РГПУ им. А. И. Герцена**, функционирующий под управлением **Grav CMS** (flat-file CMS, не использующей реляционные базы данных). Сайт развёрнут в среде контейнеризации **Docker** под оркестрацией **Kubernetes** и доступен в сети Интернет по адресу `https://ict.herzen.spb.ru`.

### 2.2. Составные части программы

Программа состоит из следующих основных частей, организованных в соответствии с файловой структурой Grav CMS.

#### 2.2.1. Ядро Grav (каталог `/system/`)

Обеспечивает базовую функциональность:
- маршрутизацию URL-запросов;
- парсинг Markdown через Parsedown;
- шаблонизацию на базе Twig;
- кэширование (Doctrine Cache);
- обработку событий (Symfony Event Dispatcher);
- интерфейс командной строки (Symfony Console).

#### 2.2.2. Пользовательская область (каталог `/user/`)

| Каталог | Назначение |
|---------|------------|
| `accounts/` | Учётные записи пользователей в формате YAML |
| `config/` | Конфигурационные файлы системы, сайта, плагинов, тем |
| `data/` | Динамические данные, генерируемые плагинами |
| `pages/` | Контент сайта в формате Markdown (17 разделов) |
| `plugins/` | Установленные плагины (более 50) |
| `themes/` | Темы оформления (активная тема — `ict`) |

#### 2.2.3. Модули интеграции с Google Cloud Platform

| Файл | Назначение |
|------|------------|
| `oauth2callback.php` | Реализация OAuth 2.0 для авторизации через Google |
| `pr_gca_save_callback.php` | Сохранение данных из Google Forms в файл `gca.json` |
| `pr_gda_save_callback.php` | Настройка соответствия страниц и папок Google Drive в файл `gda.json` |
| `pr_gd_save_callback.php` | Скачивание и конвертация файлов с Google Drive |
| `client_id.json` | Учётные данные приложения Google Cloud (проект `spb-rgpu-ict-site-2018`) |

#### 2.2.4. Инфраструктурные компоненты

| Файл | Назначение |
|------|------------|
| `Dockerfile` | Инструкция по сборке Docker-образа |
| `deploy.yaml` | Описание развертывания в Kubernetes (Ingress, Service, Deployment) |

### 2.3. Связи между составными частями

Взаимодействие компонентов описывается следующей последовательностью:

1. HTTP-запрос от пользователя поступает на **Kubernetes Ingress**, настроенный на домен `ict.herzen.spb.ru`.
2. Ingress перенаправляет трафик на **Kubernetes Service** `apps-ict` на порту 80.
3. Service направляет запрос в **Docker-контейнер** с запущенным Grav.
4. Grav Core загружает конфигурацию из `/user/config/system.yaml`, определяет активную тему (`ict`).
5. По URL определяется соответствующая страница в `/user/pages/`.
6. Markdown-контент обрабатывается плагинами группы `markdown-*`.
7. Применяется Twig-шаблон из `/user/themes/ict/templates/`.
8. При необходимости вызываются плагины (`login`, `form`, `events-manager` и др.).
9. Сформированный HTML-ответ возвращается пользователю.

**Дополнительные связи:**
- Скрипт `pr_gd_save_callback.php` использует файл `gda.json` для определения целевой папки на Google Drive.
- Скрипт `oauth2callback.php` сохраняет и обновляет токены доступа в файлы `json.txt` и `refresh.txt`.
- Скрипт `pr_gca_save_callback.php` обрабатывает вебхуки от Google Forms и сохраняет данные в `gca.json`.

### 2.4. Связи с другими программами

| Внешняя система | Протокол / API | Характер связи |
|----------------|----------------|----------------|
| Google OAuth 2.0 | HTTPS / OAuth 2.0 | Аутентификация пользователей |
| Google Forms | HTTP GET | Приём данных из форм (вебхук) |
| Google Drive API | REST API / HTTPS | Чтение файлов, конвертация форматов |
| Google Calendar API | REST API / HTTPS | Чтение событий календаря |
| Docker Engine | Container API | Запуск и управление контейнером |
| Kubernetes API | HTTP / HTTPS | Оркестрация контейнера |
| GitHub | Git / HTTPS | Возможна синхронизация через плагин `github` |
| SMTP-сервер | SMTP | Отправка уведомлений через плагин `email` |

### 2.5. Хранение данных

Ввиду использования flat-file архитектуры Grav, хранение данных осуществляется в файловой системе без применения СУБД.

| Тип данных | Формат | Расположение |
|------------|--------|---------------|
| Контент страниц | Markdown (.md) | `/user/pages/` |
| Системные настройки | YAML (.yaml) | `/user/config/` |
| Учётные записи | YAML (.yaml) | `/user/accounts/` |
| Данные Google Forms | JSON (.json) | `/user/plugins/admin/themes/grav/templates/gca.json` |
| Конфигурация Google Drive | JSON (.json) | `/user/plugins/admin/themes/grav/templates/gda.json` |
| Токены доступа OAuth | JSON (.txt) | `/php_scripts/json.txt`, `/php_scripts/refresh.txt` |
| Логи ошибок | .log | `/user/data/logerrors/` |

## 3. Настройка программы

### 3.1. Общие положения

Настройка программы производится на этапе развёртывания или при изменении условий эксплуатации. Для изменения конфигурации используются:
- файлы YAML в каталоге `/user/config/`;
- панель администратора Grav (доступна по адресу `/admin`);
- переменные окружения в среде Kubernetes.

### 3.2. Настройка на состав технических средств

#### 3.2.1. Конфигурация веб-сервера

Для работы программы требуется веб-сервер с поддержкой PHP. Рекомендуемые параметры:

| Параметр | Значение | Файл конфигурации |
|----------|----------|-------------------|
| `memory_limit` | 256M | `php.ini` |
| `upload_max_filesize` | 32M | `php.ini` |
| `post_max_size` | 64M | `php.ini` |
| `max_execution_time` | 120 | `php.ini` |

#### 3.2.2. Настройка кэширования

Для оптимизации производительности в файле `/user/config/system.yaml` настраиваются параметры кэша:

```yaml
cache:
  enabled: true
  driver: auto
  prefix: 'grav'
  lifetime: 604800
  gzip: true
```
Значения параметров:
| Параметр | Назначение | Допустимые значения |
|----------|------------|---------------------|
| `enabled` | Включение кэширования | `true` / `false` |
| `driver` | Драйвер кэширования | `auto`, `file`, `redis`, `memcached` |
| `lifetime` | Время жизни кэша (сек) | целое число |
| `gzip` | Сжатие кэша | `true` / `false` |

#### 3.2.3. Настройка обработки изображений

В файле /user/config/media.yaml задаются параметры обработки медиафайлов:
```yaml
images:
  default_image_quality: 85
  cache_all: false
  cache_perms: '0755'
  debug: false
  auto_fix_orientation: true
```

### 3.3. Выбор функций программы
#### 3.3.1. Включение и отключение плагинов
Программа включает более 50 плагинов. Управление плагинами осуществляется:

Способ 1 — через панель администратора:

Войти в панель администратора по адресу /admin
Перейти в раздел Плагины (Plugins)
Найти нужный плагин и изменить его статус (Enabled / Disabled)
Способ 2 — через конфигурационные файлы:
В файле /user/config/plugins/название_плагина.yaml установить параметр:
```yaml
enabled: true   # включить
enabled: false  # отключить
```
Основные настраиваемые плагины:
| Плагин | Назначение | Ключевые настройки |
|--------|------------|---------------------|
| `admin` | Панель администратора | `enabled`, `route`, `sidebar` |
| `login` | Аутентификация | `enabled`, `route`, `remember_me` |
| `form` | Работа с формами | `enabled`, `files` |
| `email` | Отправка почты | `enabled`, `mailer.engine`, `from` |
| `markdown-*` | Типографика Markdown | `enabled` |

#### 3.3.2. Выбор активной темы оформления
Для смены темы необходимо в файле /user/config/system.yaml изменить параметр:
```yaml
pages:
  theme: ict   # название темы из каталога /user/themes/
```
Актуальный список доступных тем находится в каталоге /user/themes/.

#### 3.3.3. Настройка мультиязычности
Для включения мультиязычности используется плагин langswitcher. Настройки задаются в файле /user/config/plugins/langswitcher.yaml:
```yaml
enabled: true
languages:
  - ru
  - en
```

### 3.4. Настройка интеграции с Google-сервисами
#### 3.4.1. Настройка OAuth 2.0 (авторизация через Google)
Для настройки авторизации необходимо:

Зарегистрировать приложение в Google Cloud Console
Получить client_id и client_secret
Сохранить учётные данные в файл client_id.json
Поместить файл в каталог /user/themes/ict/php_scripts/
Пример содержимого client_id.json:
```yaml
{
  "web": {
    "client_id": "ваш_client_id",
    "client_secret": "ваш_client_secret",
    "redirect_uris": ["https://ict.herzen.spb.ru/oauth2callback.php"]
  }
}
```

#### 3.4.2. Настройка привязки страниц к папкам Google Drive
Для привязки страницы к папке на Google Drive используется скрипт pr_gda_save_callback.php. Формат вызова:
```text
https://ict.herzen.spb.ru/pr_gda_save_callback.php?newInput_page=/путь/к/странице&newInput_folder_id=ID_папки&grav_gda_page=/страница/возврата
```
После выполнения настройки сохраняются в файл gda.json.

#### 3.4.3. Настройка импорта Google Forms
Для подключения Google Forms к сайту необходимо:

1. В Google Forms настроить отправку ответов на URL:
```text
https://ict.herzen.spb.ru/pr_gca_save_callback.php
```
2. Указать параметры запроса в соответствии с ожидаемой структурой данных
3. Данные будут сохраняться в файл gca.json

### 3.5. Настройка системы безопасности
#### 3.5.1. Управление учётными записями
Для создания нового пользователя необходимо создать файл в каталоге /user/accounts/ с именем имя_пользователя.yaml и следующим содержимым:
```yaml
state: enabled
email: user@ict.herzen.spb.ru
fullname: Имя Фамилия
title: Должность
access:
  admin:
    login: true
    super: false
  site:
    login: true
hashed_password: $2y$10$...
```
Пароль хешируется с помощью алгоритма bcrypt (например, через онлайн-генератор или командой bin/grav user).

#### 3.5.2. Настройка политик безопасности
В файле /user/config/security.yaml задаются параметры:
```yaml
xss_whitelist: []
csrf_protection:
  enabled: true
  whitelist: []
salt: 'случайная_строка'
```

### 3.6. Примеры настройки
#### Пример 1. Изменение названия сайта
Открыть /user/config/site.yaml и изменить параметр:
```yaml
title: 'Название кафедры'
```

#### Пример 2. Настройка отправки почты
В файле /user/config/plugins/email.yaml:
```yaml
enabled: true
mailer:
  engine: smtp
  smtp:
    server: smtp.yandex.ru
    port: 465
    encryption: ssl
    user: ваш_логин
    password: ваш_пароль
```

## 4. Проверка программы

### 4.1. Общие положения

Проверка программы предназначена для подтверждения её работоспособности после установки, настройки или внесения изменений. Проверка включает контрольные примеры, методы прогона и оценку полученных результатов.

### 4.2. Проверка доступности сайта

**Контрольный пример 1. Проверка HTTP-доступа**

| Действие | Ожидаемый результат |
|----------|----------------------|
| Открыть в браузере `https://ict.herzen.spb.ru` | Загружается главная страница, код ответа HTTP 200 |
| Выполнить запрос `curl -I https://ict.herzen.spb.ru` | Ответ содержит `HTTP/1.1 200 OK` |

**Метод прогона:**
```bash
curl -I https://ict.herzen.spb.ru
```

### 4.3. Проверка админ-панели

**Контрольный пример 2.** Проверка доступа к панели администратора

| Действие | Ожидаемый результат |
|----------|----------------------|
| Открыть `https://ict.herzen.spb.ru/admin` | Отображается форма входа |
| Ввести логин и пароль суперпользователя | Выполнен вход, отображается панель управления |

**Метод прогона:**
```bash
curl -L https://ict.herzen.spb.ru/admin | grep -i "login"
```

### 4.4. Проверка отображения страниц

**Контрольный пример 3.** Проверка загрузки разделов сайта

| Раздел | URL | Ожидаемый результат |
|--------|-----|----------------------|
| Главная | `/` | Отображается главная страница |
| Бакалавриат | `/baccalaureate` | Отображается информация о направлениях |
| Магистратура | `/magistracy` | Отображается информация о программах |
| Новости | `/09.events` | Отображается список мероприятий |

**Метод прогона:**

### 6.5. Коды ошибок HTTP

| Код | Название | Описание | Действия |
|-----|----------|----------|----------|
| 200 | OK | Успешный запрос | Действий не требуется |
| 301 | Moved Permanently | Страница перемещена на новый адрес | Обновить ссылки или настроить редирект |
| 302 | Found | Временный редирект | Сообщение информационное |

| 404 | Not Found | Страница не найдена | Проверить существование страницы |

| 503 | Service Unavailable | Сервис временно недоступен | Проверить состояние сервера, перезапустить контейнер при необходимости |
